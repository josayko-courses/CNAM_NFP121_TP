Ce TP est consacré à une application Java de conversion des degrés Fahrenheit en degrés Celsius.

La classe FahrenheitCelsius

Dans le répertoire du tp (qui est aussi un projet gradle), le paquetage question1 contient la classe FahrenheitCelsius.

C’est dans cette classe qu’est effectuée la conversion. Cette classe est une application car elle contient la méthode:

void main(String[] args) { //..

Complétez les 2 méthodes de cette classe

Contraintes:

  • La formule de conversion : °C = 5/9 * (°F -32)
  • Les valeurs en degrés Fahrenheit à convertir sont données sur la ligne de commande.
  • Ce sont des valeurs entières.
  • Le résultat est attendu avec un seul chiffre derrière la virgule, obtenu uniquement par le calcul. (par exemple r = ((int)resultat*10)/10.0 ).

Exécution de la classe:

  • la classe FahrenheitCelsius contient une méthode void main(String[] args), ce qui signifie qu’on peut exécuter la classe (une fois compilée) en lui passant des arguments en ligne de commande. Ces arguments sont lisible par la méthode main dans son paramètre (args) de type “tableau de chaînes de caractères”

  • pour lancer l’application: dans un terminal, passez des chaînes de caractères à la classe. Comme ceci:

    java -cp build/classes/java/main/ question1.FahrenheitCelsius 12 89

    Si vous avez correctement complété les deux méthodes, vous devriez voir apparaître les bonnes réponses (attention à traiter tous les paramètres):

    xxx°F -> yyy°C
zzz°F -> ttt°C

Technique Java:

  • Les arguments de la ligne de commande sont des chaînes (String), donc pour effectuer la conversion il faut les convertir en entiers (cf. static int parseInt(String s) de la classe Integer du paquetage java.lang).
  • Pour les tableaux, le champ length contient le nombre d’éléments du tableau (ATTENTION le premier indice d’un tableau est 0).
  • Attention : la division entre 2 entiers est une division entière. Exemple 5/9 = 0 et 9/5 = 1
  • Il sera peut être nécessaire de modifier le chemin d’accès aux exécutables du JDK, sous Windows set PATH="c:/chemin vers le jdk/bin";%PATH% ou bien modifier la variable d’environnement PATH.
  • vous devriez également pouvoir lancer l’application (et lui passer des paramètres) depuis votre IDE.

La gestion de l’exception NumberFormatException

“runtime” Exceptions :

Dans le paquetage “question2”, on trouve une autre version de la classe FahrenheitClesius.

Développez une nouvelle version de cette classe en reprenant la conversion de la question 1 et en ajoutant la gestion de l’exception NumberFormatException.

L’exception NumberFormatException est levée lors d’une mauvaise entrée sur la ligne de commande i.e. d’une entrée autre qu’une chaîne convertible en int Exemple : java -cp build/classes/java/main/ question1.FahrenheitCelsius "10" "ZZ" "100" engendre :

Exception in thread "main" java.lang.NumberFormatException: For input string: "ZZ"
    at java.base/java.lang.NumberFormatException.forInputString(NumberFormatException.java:65)
    ...

IHM

Dans le paquetage question3, complétez l’“IHM Fahrenheit” fournie en utilisant la fonction de conversion de la question, cette IHM doit avoir un comportement analogue à cette interface. Celle-ci s’exécutera facilement dans votre environnement en déclenchant la méthode de classe main.

IHM de référence

En exécutant l’interface de “référence” vous pouvez voir le comportement attendu de l’IHM, pour cela il vous faut télécharger le ihm.jar, puis dans une console:

Il sera peut être nécessaire de modifier le chemin d’accès à cet exécutable, sous Windows tp2> set PATH="c:/chemin vers le jdk/bin";%PATH% ou bien modifier la variable d’environnement.

Prise en compte de l’exception NumberFormatException

Ajoutez à l’IHM la prise en compte de l’exception NumberFormatException: si cette exception est levée la sortie mentionnera le libellé “error !”, comme le montre cette copie d’écran.
