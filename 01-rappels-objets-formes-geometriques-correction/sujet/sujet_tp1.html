Par exemple certains tests exigent que vos tests contiennent un certain nombre d’appels à des méthodes de AuditeurCNAM.

Conseils valables pour la suite du semestre:

Conseil: lisez bien le message d’erreur du test, en général il donne des informations sur ce qui était testé.

Astuce 1: Ensuite consultez le code du test (si il est disponible, voir plus bas). En général si vous lancez les tests via votre IDE vous pouvez double cliquer sur le rapport du test dans l’IDE et cela ouvre le fichier à l’endroit où le test a échoué.

Astuce 2: En général les IDE permettent de lancer un seul test à la fois, voire un seul fichier de test à la fois. Trouvez comment faire, cela vous permettra d’être plus productif: certains projets (y compris en NFP121) ont des tests qui sont assez longs et qui exigent parfois que vous ne touchiez ni au clavier ni à la souris, pouvoir ne lancer qu’un seul test à la fois est donc vraiment utile.

Remarque: Les fichiers markXXX.java sont les fichiers contenant les tests d’évaluation de votre TP. Cela ne veut pas dire que le TP est nécessairement noté, seulement que ce sont les tests du profs et que si vous voulez écrire des tests il faut le faire dans un autre fichier.

Remarque: Dans ce tp les tests de notation markXXX.java sont donnés en clair. Parfois ce ne sera pas le cas car écrire des tests pertinents (dans un autre fichier) sera un des exercices du tp. Les tests de notations seront alors donnés sous la forme d’un fichier testlib/ZTestOfficial.jar contenant les fichiers .class (sous forme déjà compilée donc). Vous pourrez donc lancer les tests du prof, voir les résultats et les messages d’erreur, mais pas le code.
